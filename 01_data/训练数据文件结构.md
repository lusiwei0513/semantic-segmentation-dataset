# 可用于训练的图像数据文件结构

> 入口目录：`01_data/`（多为联接，实体多在 `02_baselines_unet_deeplab/`）  
> 正式对比划分：**fold_0** = train 133 / val 18 / test 38（按 `vehicle_id`）

---

## 1. 一句话：训练到底读什么

| 用途 | 必读路径 | 数量 |
|------|----------|------|
| 正视分割（UNet-seg / DeepLab / SegFormer） | `data_unet/front/{images,masks}` + `data/front/splits/fold_0.json` | 图/mask 各 189 |
| 正视 UNet-KP（另加鼻尖） | 上面 + `data/front/processed/keypoints/*.json` | keypoints 189 |
| 侧视分割（UNet / DeepLab / SegFormer） | `data_unet/side/{images,masks}` + `data/side/splits/fold_0.json` | 图/mask 各 189 |
| 仅打开 LabelMe 改标注 | `labelme_workspace/{front,side}`（图+json 成对） | 各 189 对 |

**不要**用 `prepared_*` 当正式对比数据源（那是早期全量包）；**不要**指望 `prepared_*` 里有 LabelMe 的 `.json`。

## 2. 推荐目录树（训练就绪）

```
01_data/                              ← 建议从这里看
│
├── data_unet/                        ★ 主训练：图 + 类别 mask
│   ├── front/
│   │   ├── images/                   # RGB（stem 如 front_xxxx）
│   │   ├── masks/                    # 单通道 PNG，像素值=类别 id
│   │   └── classes.json              # 0 bg, 1 body, 2 windshield, 3 nose_tip
│   └── side/
│       ├── images/
│       ├── masks/
│       └── classes.json              # 0 bg, 1 body, 2 windshield, 3 bogie, 4 door
│
├── data/                             ★ 划分 + 正视关键点
│   ├── front/
│   │   ├── splits/
│   │   │   ├── fold_0.json           # 正式对比用
│   │   │   └── fold_1.json … fold_4.json
│   │   └── processed/
│   │       └── keypoints/            # UNet-KP：每图一个 tip json
│   └── side/
│       └── splits/
│           └── fold_0.json …
│
├── labelme_workspace/                ★ 标注软件打开用（非训练直接读）
│   ├── front/                        # 图 + 同名 .json
│   └── side/
│
├── prepared_front/ / prepared_side/  # 早期转 mask 包（可作备份/预览）
├── split_raw/                        # 原始裁切图（溯源，一般不直接训）
└── 训练数据文件结构.md               # 本文件
```

实体等价路径（配置里常用相对路径）：

```
02_baselines_unet_deeplab/
├── data_unet/{front,side}/
├── data/{front,side}/splits/
└── data/front/processed/keypoints/
```

SegFormer（`03_segformer_split`）通过 `data_front` / `data_side` / `splits_*` **联接**到同一套数据，不另存一份主数据。

## 3. 单张样本如何对应

以正视为例（侧视把 `front` 换成 `side`，无 tip）：

| 文件 | 作用 |
|------|------|
| `data_unet/front/images/front_<hash>.jpg` | 训练输入图 |
| `data_unet/front/masks/front_<hash>.png` | 像素级 GT |
| `data/front/splits/fold_0.json` 中的 id | 决定属于 train/val/test |
| `data/front/processed/keypoints/front_<hash>.json` | 仅 KP：鼻尖坐标 |

`fold_0.json` 结构示意：

```json
{
  "fold": 0,
  "train": ["front_xxxx", "..."],
  "val": ["..."],
  "test": ["..."]
}
```

---

## 4. 从标注到可训练（数据流）

```
labelme_workspace 中的 图 + .json
        ↓  prepare_masks / 既有脚本
data_unet/*/images + masks
        ↓  固定 fold_0.json
train / val / test 加载
```

- **标注层**：`labelme_workspace`（LabelMe / X-AnyLabeling）  
- **训练层**：`data_unet` + `splits`（+ 正视 `keypoints`）  
- **原始层**：`split_raw`（备份溯源）

## 5. 配置里怎么写（对照）

| 任务 | `prepared_dir` / 图像根 | `fold_json` | 额外 |
|------|-------------------------|-------------|------|
| 正视 UNet-KP | `../data_unet/front` | `../data/front/splits/fold_0.json` | `keypoints_dir` |
| 正视 UNet / DeepLab | `../data_unet/front` | 同上 | — |
| 侧视 UNet / DeepLab | `../data_unet/side` | `../data/side/splits/fold_0.json` | `image_size: [384,1536]`（宽幅） |
| SegFormer | 工程内 junction → 同上 processed/splits | `splits_front` / `splits_side` | — |
